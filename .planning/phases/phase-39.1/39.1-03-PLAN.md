<objective>
Close G3-BOARD-SWITCH-CLEAN (operator-visible "partial align mode" + "spurious dirty flag" during board switches).

Root cause #1 (HIGH confidence per RESEARCH §G3): the `onProjectionProfileChange` handler at `output-align-mode-loader.js:807-816` does `await activate()` async. `activate()` contains a dynamic `import("boot-handle-ui.js")` (line 525) that yields control. Meanwhile, the dashboard-side `switchBoard → autoLoadRememberedProjectionProfile → applyDefaultAndCaptureSnapshot()` emits `broadcastGridSnapshot({force:true})`. The Pi /output/ receives `align-grid-snapshot`, applies the new grid via `_applyAlignGridSnapshot`, and immediately calls `rebuildHandleElements/positionHandles/drawLines`. The OLD `_currentBootHandle` is still bound to the OLD boardId, so handles render with the NEW grid but `renderRoomOverlay` reads polygons for the OLD board — operator sees handles without overlay lines.

Root cause #2 (HIGH confidence trace, [ASSUMED] until probe confirms — that is what 39.1-01 Task 5 landed): Hypothesis C — HANDLE_UI's `_refreshAlignToolbarVisual()` reads stale `_loadedProfileSnapshot` after the remote-pushed grid changes the comparison state, so `isDirty()` returns true and `_postAlignModeDirtyToServer(true)` posts a spurious dirty=true to the dashboard.

Fix (per RESEARCH §G3 sub-G3a + sub-G3b):
- Sub-G3a: synchronous `_currentBootHandle.stop()` + body-class removal BEFORE any await in onProjectionProfileChange handler. Add `_activationToken` counter; check token after every await in `activate()`.
- Sub-G3b: Add `captureCurrentAsLoadedSnapshot(name)` to runtime-projection-profile-persistence.js. Call it from output-live-sync.js's `_applyAlignGridSnapshot` after `restoreGridSnapshot` so server-pushed snapshots become the new clean baseline.
