## Exact format of the G3-PROBE log line

For 39.1-3 to grep during operator reproduction, the probe emits a single
`console.warn` of the form:

```
[G3-PROBE] dirty=true POST initiated boardId=<id> loadedProfile=<name|(none)> hasSnapshot=<true|false> grid=<rows>x<cols>
stack:
<Error("G3-PROBE").stack>
```

Grep pattern (Plan 39.1-3 should use this):

```
grep -E "^\[G3-PROBE\] dirty=true POST initiated"
```

The stack trace contains the JS call site (typically a HANDLE_UI handler
or the dirty broadcaster) so 39.1-3 can confirm whether
`_refreshAlignToolbarVisual` (Hypothesis C) is the actual caller during
a real board-switch reproduction.

Probe is **permanent** per plan §<behavior>: only fires on `dirty=true`
POSTs (already 100 ms server-rate-limited via T-27-03) so overhead is
negligible in production.

## helpers.switch_board mechanism

`switch_board(page, board_id)` POSTs to `/api/live/command` with
`mutationType: "context-update"` and `payload: {selectedBoard: board_id}`,
then polls `/api/live/snapshot` for `snapshot.selectedBoard === board_id`
(5s timeout, 100ms interval).

**Why not the plan's literal `/api/live/board-switch`?** That endpoint
does NOT exist in `server.mjs` (verified by `grep -n "/api/live/board-switch"
server.mjs` → empty result). `LIVE_MUTATION_TYPES` at server.mjs:149-168
shows no `board-switch` entry either; `context-update` is the canonical
board-switch wire format (see server.mjs:1189 `else if (mutationType ===
"context-update")` branch).

## Deviations from Plan

### [Rule 3 — Blocking issue] Sync tests instead of `@pytest.mark.asyncio`

- **Found during:** Task 1 (helpers.py authorship).
- **Issue:** The plan specified async helpers + `@pytest.mark.asyncio` test
  decorators. The existing `test/live-e2e/conftest.py` uses
  `sync_playwright()` (not async); `pytest-asyncio` is NOT installed
  (`import pytest_asyncio` → ModuleNotFoundError). Async tests would fail
  at collection.
- **Fix:** Wrote sync versions of all 5 helpers + 3 tests, operating on
  the existing `live_server` + `page` (sync Playwright) fixture chain.
  All helper names, semantics, return shapes, and assertion thresholds
  preserved per plan. Async signatures can be reintroduced cheaply
  later by wrapping each body in `async def` + `await page.evaluate(...)`
  once pytest-asyncio is added.
- **Files modified:** all 4 new test/live-e2e files (sync style).
- **Commits:** 7bff5cc, 387f855, af351a1, 503ecef.

### [Rule 3 — Blocking issue] `/api/live/board-switch` does not exist; use `context-update`

- **Found during:** Task 1 (switch_board helper authorship).
- **Issue:** Plan specified `helpers.switch_board` should POST to
  `/api/live/board-switch` with a fallback to
  `emitLiveMutation('board-switch', ...)`. Neither exists:
  - `grep -n "/api/live/board-switch" server.mjs` → empty.
  - `LIVE_MUTATION_TYPES` at server.mjs:149-168 has no `board-switch` entry.
- **Fix:** Use `/api/live/command` with `mutationType: "context-update"` and
  `payload: {selectedBoard: <id>}` — the canonical wire format for board
  selection (verified at server.mjs:1189 + the selectedBoard reads at 500-547).
  Semantics are identical from the /output/ runtime's perspective.
- **Files modified:** test/live-e2e/helpers.py.
- **Commit:** 7bff5cc.
