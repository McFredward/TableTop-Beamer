## Deviations from Plan

### [Rule 2 — Critical operational behavior] G3 live-e2e test status interpreted as DEFERRED, not FAILED

- **Found during:** Task 1 regression matrix run, attempts 1 and 2
  on `test_phase39_1_g3_board_switch_clean.py`.
- **Issue:** The test fails at the very first step
  (`trigger_align_mode_on` → `wait_for_function` 10s timeout)
  because two test-infrastructure limitations stack:
  (a) `helpers.trigger_align_mode_on` uses `mutationType:
  "align-toggle"` which DOES NOT propagate `alignMode` to the
  consumer-side `liveSync.getAlignMode()` (verified by direct probe:
  `/api/live/snapshot` returns `alignMode=null` after the mutation,
  and `window.TT_BEAMER_OUTPUT_LIVE_SYNC.getAlignMode()` stays
  false). The canonical wire format the consumer reads is
  `context-update` with `payload.alignMode=true` (output-live-sync.js
  line 104: `if (typeof snap.alignMode === "boolean")`).
  (b) Even after switching to `context-update`,
  `[data-grid-handle]` elements never mount in the headless
  consumer view because `getRoomSourcePoints` returns null absent
  an operator session that has loaded a board+profile (verified by
  30+ console warnings:
  `[align-loader] getRoomPoints helper threw — falling back:
  Cannot read properties of null (reading 'getRoomSourcePoints')`).
  The body class flips correctly (`align-mode-active` appears) and
  overlay polygons render (66 polygons appear), but the
  projection-grid-handles do not.
- **Disposition:** This is a TEST INFRASTRUCTURE limitation, NOT a
  regression of the Phase 39.1-03 G3 fix. The 39.1-03 SUMMARY
  (line 283) ALREADY documented this exact DEFERRAL: "G3 RED test
  ... | DEFERRED | Requires live SSR + Chromium tab; sandbox has
  neither. Collects cleanly. 39.1-04 operator UAT exercises
  end-to-end." Per the auto-mode operator directive, a documented
  DEFERRAL with prior cross-reference is NOT an automated regression
  failure — closure proceeds and the operator visual UAT carries
  the acceptance.
- **Fix:** Recorded the test as DEFERRED in
  `.planning/phases/phase-39.1/39.1-VERIFICATION.md` regression
  matrix and added a detailed § "G3 Test Infrastructure Limitation"
  section explaining the failure mode, the static verification of
  the G3 fix (commits + greppable surface), and the deferral
  rationale.
- **Open follow-up:** Defer G3 test refactor (programmatic operator
  state bootstrap via `/api/live/command` with
  `mutationType: "projection-profile-load"`, OR run the test against
  the dashboard `/` view where handles always mount when align mode
  is active) to Phase 40+ as test-infrastructure improvement.
- **Files modified:** none (test file itself unchanged; only
  documentation in 39.1-VERIFICATION.md and this SUMMARY).
