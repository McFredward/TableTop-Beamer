## Code-Trace Argument for Hypothesis C

The plan's `<context>` cites RESEARCH §G3 lines 480-490 and `[ASSUMED]` flag at line
589. The trace eliminates A and B and points to C:

**Hypothesis A (dashboard-side dirty=true broadcast):** Dashboard's
`applyDefaultAndCaptureSnapshot` (runtime-projection-profile-persistence.js:702-726)
captures `_loadedProfileSnapshot = _gridStateApi.snapshotGridState()` BEFORE calling
`_recomputeAndNotifyDirty()`, so `isDirty()` (line 205-217) returns false (snapshot
just captured == current grid). `_dirty` stays false. `_recomputeAndNotifyDirty`'s
listener-fan-out at lines 219-226 is short-circuited by the `if (next === _dirty)
return` guard at line 221 → no listener fires → no dirty POST. **Path A eliminated.**

**Hypothesis B (Pi /output/-side direct emission from _applyAlignGridSnapshot):**
`output-live-sync.js:175-260` calls `gs.restoreGridSnapshot(...)` then
`ui.rebuildHandleElements/positionHandles/positionRotateHandles/drawLines`.
`restoreGridSnapshot` (in grid-state.js — Phase 38 W12 left this unchanged) does NOT
call `notifyDirtyChanged()`. The four UI redraws in handle-ui.js also don't call
`notifyDirtyChanged()` — `grep -n notifyDirty src/app/runtime/viewport/runtime-projection-handle-ui.js`
shows the only calls are from drag-event handlers, not redraw/position/draw functions.
The W4 drain path in output-align-mode-loader.js:647-707 reaches `_applyAlignGridSnapshot`
via `applyPendingGridSnapshot()` — same path, no extra dirty emission.
**Path B eliminated.**

**Hypothesis C (HANDLE_UI's toolbar-visual refresh reads stale snapshot):** When
boot-handle-ui activates (line 525 of output-align-mode-loader.js), its subscription
to `_offAlignModeChange = liveSync.onAlignModeChange(...)` fires `HANDLE_UI.onAlignModeChange(true)`
which triggers `showHandles()` → `_refreshAlignToolbarVisual()` → reads
`getCurrentBoardId()` → calls `persistence.isDirty()`. `isDirty()` at line 205-217
reads `_loadedProfileSnapshot` (the OLD board's snapshot — the new board's
`align-grid-snapshot` arrived via `_applyAlignGridSnapshot`, mutated the live grid,
but did NOT update `_loadedProfileSnapshot`). The current grid != the stale loaded
snapshot → `isDirty()` returns TRUE → addDirtyListener fires →
`_postAlignModeDirtyToServer(true)` posts dirty=true. **This is the predicted path.**

The fix: when a remote `align-grid-snapshot` is applied, re-baseline
`_loadedProfileSnapshot` so `isDirty()` returns false after the apply. This makes
"remote snapshot arrival" semantically equivalent to "loaded the same profile" from
the perspective of dirty tracking — which is the correct semantic per RESEARCH §G3
R3.3 (Pi /output/ is a consumer, not a producer of authoritative profiles).
