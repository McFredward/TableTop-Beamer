# Phase 8 Workspace

Planning and execution workspace for multi-play-area support and board image import.

- `PLAN.md`: target architecture, migration strategy, and scope boundaries for Phase 8.
- `BACKLOG.md`: epics and story mapping for multi-area geometry and image-upload board import.
- `TASKS.md`: prioritized implementation tasks; Plan 8-1 is the first execute-ready wave.
- `ACCEPTANCE.md`: mandatory quality gates for geometry semantics, migration safety, upload/security, and non-regression.
- `RISKS.md`: key delivery risks with mitigations.
- `EXECUTE.md`: binding execution order and gate rules.
