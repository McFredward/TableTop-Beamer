# Plan 18-1: Dashboard UX Overhaul

## Objective
Restructure the Dashboard view for intuitive operation. Quick Mode becomes the primary interaction pattern (top position), status noise is eliminated, empty states guide new users, and the room animation section uses progressive disclosure. Button labels become self-explanatory (Select/Start/Stop/Clear).

## Tasks

### T1 — Quick Mode Promotion & Renaming
- Move the Quick Mode panel (`#quick-mode-panel`) to be the FIRST section after the view-switch nav in Dashboard
- Rename heading: "Selection mode" → "Quick Mode"
- Rename buttons: "Activate" → "Start", "Deactivate" → "Stop", "Clear room" → "Clear"
- Update JS: button IDs stay the same, but any text references in JS status updates must reflect new labels
- Status text: replace static "Quick mode: OFF" with contextual messages:
  - OFF: "Select a room on the board"
  - Start: "Tap a room to start its animation"
  - Stop: "Tap a room to stop its animation"
  - Clear: "Tap a room to clear all its animations"
- Keep `quick-mode-off` button as "Select" (unchanged)

### T2 — Room Animation Section Simplification
- Rename `<details id="room-options"><summary>More options</summary>` → `<summary>Advanced overrides</summary>`
- Move `dashboard-default-animation` checkbox OUT of the details element — place it right after the Start button (visible by default)
- Stagger options (`room-stagger-start`, `room-stagger-offset`): wrap in a container that is hidden (`style="display:none"`) unless target is a cluster — JS already knows target type
- Transform fieldset inside details: wrap in its own `<details>` for double-nesting (collapsed by default)

### T3 — Status Lines Cleanup (Dashboard + Settings)
Remove these static/noise status lines from index.html:
- "Room animations run in hold mode by default until stopped." (in room animation section)
- "Touch safety: `Clear All` requires an explicit double confirmation." (in active animations section)
- "Priority section: runtime status with direct Stop/Edit actions." (in active animations section)
- "Note: Download export is a secondary fallback. Primary path remains API save." (in board catalog)
- "Board source: checked at startup" (`#zones-status`)
- "API diagnostics: checked in save preflight" (`#api-diagnose-status`)
- Settings header lead text: shorten to "Configure board, animations, and system settings."
- "Calibrate hit areas and room geometry..." → remove entirely
- Remove `#settings-subtab-status` line ("Settings focus: Board & Geometry")
- Keep dynamic status lines: `#room-selected`, `#board-status`, `#board-zoom-status`, `#quick-mode-status`, `#trigger-feedback`, `#audio-status`

### T4 — Empty States & Micro-Onboarding
- Dashboard header: shorten lead text to "Control room animations and effects during your game."
- `#room-selected` default text: "No room selected — click a room on the board" (instead of "Selected room: click a room polygon on the board")
- Running animations empty state (`running-empty` class): update text to "No active animations"
- Settings header: shorten to "Configure board, animations, and system settings."

### T5 — Active Animations Panel Polish
- Live Editor: wrap `#live-editor-transform` fieldset in `<details>` so transform is collapsed by default
- Running items: reduce padding from `0.62rem` to `0.5rem`, reduce gap from `0.48rem` to `0.35rem`
- Running actions buttons: ensure min-height stays at 2.9rem for touch targets

## Must-haves
- Quick Mode panel is the first dashboard section after view-switch
- Button labels are Select/Start/Stop/Clear
- All listed static status lines are removed
- No functionality removed — all existing features still work
- Quick mode JS logic unchanged (only labels/text)
