# Plan 18-3: Polygon Workflow & Undo/Redo

## Objective
Make polygon creation accessible directly from the board (right-click/long-press context menu or dedicated button). Visually separate Room and Play Area creation to prevent accidental mix-ups. Add undo/redo (Ctrl+Z / Ctrl+Shift+Z) for polygon editing operations with visible buttons.

## Tasks

### T10 — Direct Polygon Creation on Board
- Add a context menu for the board overlay (`#room-overlay`):
  - Desktop: right-click on empty area (no room polygon hit) → shows a small popup menu with "Add room here"
  - Mobile: long-press (>500ms) on empty area → same popup menu
  - Menu is a simple positioned `<div>` with one button, dismissed on click-outside or Escape
  - Clicking "Add room here" creates a new room with a default hexagon centered at the click/touch position
  - The new room gets an auto-generated name like "Room N" (next available number)
  - After creation: auto-select the new room in the polygon editor
- Visual separation of Room vs Play Area create buttons in Settings:
  - Room create section: add a subtle left-border accent in green/teal (matching room zone color)
  - Play Area create section: add a subtle left-border accent in blue/purple (matching play area color)
  - Ensure these two sections are NOT adjacent — cluster management sits between them
- Implementation:
  - Add `contextmenu` event listener on `#room-overlay` (prevent default)
  - Add `pointerdown` + timer for long-press detection on mobile
  - The context menu div is appended to `.app-shell` (not inside the stage, to avoid transform issues)
  - Position using clientX/clientY from the event
  - Convert click position to normalized room coordinates using the existing stage-to-SVG coordinate transform

### T11 — Undo/Redo System
- Create a new module `src/app/runtime/polygon-editor/runtime-polygon-undo.js`:
  - Exports: `pushUndoState(description)`, `undo()`, `redo()`, `canUndo()`, `canRedo()`, `clearUndoStack()`
  - Internal: `undoStack[]` and `redoStack[]`, max 50 entries each
  - Each entry stores: `{ description, roomStates }` where roomStates is a snapshot of all room polygons + names
  - `pushUndoState()`: captures current room geometry state, pushes to undoStack, clears redoStack
  - `undo()`: pops from undoStack, pushes current state to redoStack, restores popped state
  - `redo()`: pops from redoStack, pushes current state to undoStack, restores popped state
  - `clearUndoStack()`: empties both stacks (called on board switch)
- Add undo state capture points in existing code:
  - Before vertex drag completes (in drag end handler)
  - Before vertex insert/delete
  - Before room create/delete
  - Before room rename
- Add UI:
  - Two buttons in the polygon editor area: "Undo" and "Redo" (in a button-grid-2)
  - Buttons are disabled when respective stack is empty
  - Buttons update their disabled state after every undo/redo/push operation
- Add keyboard shortcuts:
  - `Ctrl+Z` → undo (only when Settings view is active with Board subtab)
  - `Ctrl+Shift+Z` → redo (same condition)
  - Listener on `document`, checks active view before acting
  - Prevents default browser undo behavior when shortcut is consumed
- Add script tag for new module in index.html
- Wire into runtime-orchestration.js init

## Must-haves
- Right-click on empty board area shows "Add room here" menu
- Long-press on mobile shows same menu
- New room is created at click position with auto-name
- Undo/Redo works for vertex drag, insert, delete, room create, room delete, room rename
- Ctrl+Z and Ctrl+Shift+Z keyboard shortcuts work
- Visible Undo/Redo buttons in polygon editor
- Undo stack clears on board switch
- Room and Play Area creation buttons are visually distinguished
