# Plan 18-2: Settings UX Overhaul

## Objective
Consolidate Settings panels to reduce scrolling, rework the animation create/edit workflow for clarity, update terminology from technical to user-friendly, and clean up navigation labels.

## Tasks

### T6 — Settings Panel Consolidation
- Merge "Board catalog + output" panel and "Board zoom" panel into ONE panel titled "Board Setup"
  - Board select, import controls, export/import config buttons, zoom controls, zoom status — all in one panel
  - Remove duplicate panel borders/headings
- Merge "Room management" panel and "Room polygon editor" panel into ONE panel titled "Room Editor"
  - Room create/delete at top, then polygon editing controls below
  - Room name rename field stays in the polygon section area
  - Freeze checkbox stays
- Keep "Cluster management" as its own panel
- Keep "Play Area editor" as its own panel — but rename to "Play Area"
- Clean up Settings status lines:
  - Remove `#polygon-editor-status` ("Polygon editor: select a room")
  - Remove `#room-management-status` ("Room management: ready")
  - Remove `#cluster-management-status` ("Cluster management: ready")
  - Remove `#ship-polygon-editor-status` ("Play Area editor: ready")
  - Remove `#global-defaults-status` ("Global Defaults: not saved yet")
  - Remove `#align-mode-status` ("Align-Mode: OFF")
  - Keep `#board-status` (dynamic: "Active board: X")
  - Keep `#board-zoom-status` (dynamic: "Zoom: 100%")

### T7 — Animation Create/Edit Workflow Rework
For each animation section (Room, Inside, Outside):
- Add a visual mode indicator showing current state:
  - When an existing animation is selected in dropdown: show "Editing: {name}" as a small badge/label
  - When creating new: show "Creating new animation" badge
- Move "Delete" button: only visible when an existing animation is selected (not during create)
- Create flow: name input + create button stay together, clearly separated from edit controls
- Ensure consistent field ordering across all three sections: Name/Create → Dropdown → Type → Source → Sound → Defaults

### T8 — Animation Settings Terminology
- All three animation sections (Room, Inside, Outside):
  - "Asset type" label → "Type"
  - Options: "Coded" → "Effect", "GIF" stays "GIF", "MP4" → "Video"
  - "Asset reference" input → hide it entirely (set `display:none`), keep the DOM element for backward compat
  - "Resource asset picker" label → "Source"
  - Sound label stays "Sound" (already clear)
- Update JS: any code that reads/writes the select option values must still use "coded"/"gif"/"mp4" internally — only the display text changes
- Use `<option value="coded">Effect</option>` pattern (value stays, display changes)

### T9 — Settings Header & Navigation
- Subtab button labels:
  - "Board & Geometry" → "Board"
  - "Animations" stays "Animations"
  - "System & Performance" → "System"
- Settings header `<h1>`: stays "Settings Workspace"
- Settings header lead: "Configure board, animations, and system settings."
- Remove `#settings-subtab-status` paragraph entirely from DOM

## Must-haves
- Board Setup is one consolidated panel (was 2)
- Room Editor is one consolidated panel (was 2)
- Animation create vs edit state is visually indicated
- Terminology uses Effect/GIF/Video (not Coded/GIF/MP4) in display
- Internal values (coded/gif/mp4) unchanged
- All existing Settings functionality preserved
