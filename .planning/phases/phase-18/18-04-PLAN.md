# Plan 18-4: Mobile-First Polish

## Objective
Optimize the mobile experience for in-game use. Quick Mode becomes easily reachable without scrolling, running animations are compact, and Settings subtabs use horizontal pill navigation.

## Tasks

### T12 — Mobile Quick Mode Optimization
- Mobile zone switch labels: "Trigger" → "Control", "Manage running" → "Active"
- Quick Mode buttons on mobile: increase min-height to 3.5rem, ensure 48px touch targets
- Quick Mode status should remain visible during scroll on mobile

### T13 — Mobile Running Animations
- Running items on mobile: more compact layout
- Stop buttons on mobile: larger touch target, prominently colored
- Live Editor on mobile: reduce vertical space, most important sliders (opacity, intensity, speed) first

### T14 — Mobile Settings
- Settings subtab switch on mobile: horizontal row of pill-shaped buttons (not vertical stack)
- Settings panels on mobile: reduce padding for more content visibility

## Must-haves
- Mobile zone switch says "Control" / "Active"
- Quick Mode buttons have min 48px touch targets on mobile
- Settings subtabs are horizontal pills on mobile
