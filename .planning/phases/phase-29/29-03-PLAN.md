dead audio-mapping UI plumbing. Per 29-AUDIT.md §F3 (RESEARCH §F3): the map is
REDUNDANT to per-animation `animation.soundAssetRef` (set authoritatively by
`runtime-animation-factory.js:54-55`), and the audio-mapping UI panel itself
was deleted from `index.html` in a prior phase — only the dead JS plumbing
remains.

**Critical sequencing:** This plan removes the SOURCE-side reads of
`state.animationSoundMap`. The DISK migration (lossless copy of remaining map
values into per-animation `soundAssetRef` slots) happens in Wave 3 (29-05),
which BOOTS the server and runs the migration BEFORE the cleaned-up source
code first reads any board JSON.

Why this order is safe: the per-animation `soundAssetRef` is already
authoritative (factory writes it on creation). The map was only ever a
fallback. Removing the fallback means animations without a per-animation ref
(if any exist) play silently — but Wave 3's migration guarantees ALL such
animations get their refs populated before the cleaned source first runs.
