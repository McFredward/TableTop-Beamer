# Plan 19-3: Grid Mesh Warp for Projection Mapping

## Objective
Add a variable-density grid mesh warp to /output during align mode. The grid allows local distortion corrections by dragging grid intersection points. Works on top of the existing 4-corner perspective transform. Default 4x4 grid, right-click to add/remove grid lines.

## Tasks

### T9 — Offscreen Canvas + Grid Mesh Rendering
- In the draw loop on /output, render to an offscreen canvas instead of directly to the visible canvas
- After rendering, redraw the offscreen canvas onto the visible canvas through a deformed grid mesh:
  - Each grid cell maps a source rectangle (from offscreen) to a destination rectangle (on visible canvas)
  - Source rectangles are evenly spaced based on grid line positions in normalized space
  - Destination rectangles are based on the user-adjusted grid point positions
- Grid state: array of horizontal line positions (0-1 normalized) and vertical line positions (0-1 normalized)
- Default 4x4: 3 horizontal interior lines at 0.25, 0.5, 0.75 and 3 vertical interior lines at 0.25, 0.5, 0.75
- Grid point displacements stored as dx/dy offsets per intersection point
- Performance: only use offscreen rendering on /output when grid has non-zero displacements

### T10 — Grid Overlay UI (Align Mode on /output)
- During align mode on /output, render grid lines and draggable intersection handles
- Grid lines: thin semi-transparent lines (white/cyan) connecting grid points
- Intersection handles: small circles at each grid crossing point, draggable
- Dragging a handle displaces that grid point, deforming the adjacent cells
- Edge and corner handles: grid points on the border move along their edge only
  (top/bottom edge points: horizontal only, left/right edge points: vertical only)
- Corner grid points are fixed (they correspond to the 4-corner projection handles)
- Use a canvas overlay for lines + DOM elements for handles (same pattern as corner handles)

### T11 — Right-Click Context Menu for Grid Lines
- Right-click on /output during align mode shows a context menu:
  - If clicking near a grid line: "Remove this line"
  - If clicking in empty space: "Add horizontal line here" / "Add vertical line here"
- Adding a line: inserts at the clicked position (normalized 0-1), splits affected cells
- Removing a line: removes the line and merges adjacent cells
- Minimum: 1 interior line per axis (can't remove all lines)
- Grid line positions and displacements update accordingly
- Reuse the board-context-menu CSS pattern from Phase 18

### T12 — Persistence + Reset
- Store grid state in localStorage: `tt-beamer.projection-grid`
  - { horizontalLines: [0.25, 0.5, 0.75], verticalLines: [0.25, 0.5, 0.75], displacements: { "row-col": { dx, dy } } }
- Load on init, save after every adjustment
- Reset: context menu option "Reset grid" sets all displacements to zero and restores default 4x4

## Must-haves
- Grid mesh warp works on /output, deforming rendered content per-cell
- Default 4x4 grid with draggable intersection points
- Right-click to add/remove grid lines
- Grid displacements persist in localStorage across page refresh
- Works together with 4-corner perspective transform
- No performance impact on Dashboard or /output when grid has zero displacements
- Grid overlay only visible during align mode
