# Plan 19-2: Projection Mapping (4-Corner Warp)

## Objective
Add a 4-corner projection mapping system to /output that allows the user to drag corner handles to warp the projected image for perfect beamer alignment. Similar to scripts/projection_mapper.py but implemented in the browser using CSS 3D transforms.

## Tasks

### T4 — Projection Mapping Module
- Create new module: `src/app/runtime/viewport/runtime-projection-mapping.js`
- IIFE + init(ctx) pattern matching existing codebase
- State: 4 corner positions stored as `{ topLeft, topRight, bottomRight, bottomLeft }` each with `{ x, y }` in percent (0-100)
- Default: corners at screen edges (0,0), (100,0), (100,100), (0,100)
- Compute CSS `matrix3d()` transform from 4 corner positions using perspective projection math
- Apply the transform to the `.stage` element on /output only
- Outside the warped area: CSS `background: #000` on body (already black on /output)

### T5 — Corner Drag Handles on /output
- Only visible during Align Mode on /output (OUTPUT_ROLE_FINAL)
- Render 4 draggable corner handles as absolutely positioned divs overlaying the viewport
- Each handle: 24px circle, red color (matching projection_mapper.py), labeled 1-4
- Drag behavior: pointerdown + pointermove + pointerup on each handle
- Connect corners with red border lines (CSS outline or SVG overlay)
- When dragging: update corner position in real-time, recompute and apply CSS matrix3d()

### T6 — Arrow Key Fine-Tuning
- During Align Mode on /output, arrow keys nudge the active/hovered corner:
  - Arrow keys: 1px movement
  - Shift + Arrow keys: 10px movement
  - Tab: cycle active corner (1→2→3→4→1)
- Active corner highlighted with different color/size
- Only active on /output when align mode is on
- Keyboard listener with proper focus management (no conflict with other shortcuts)

### T7 — Persistence
- Save corner positions to server via existing config mechanism:
  - Store in `global-defaults.json` under `projectionMapping: { corners: { topLeft, topRight, bottomRight, bottomLeft } }`
  - Save automatically when align mode is turned off (debounced)
  - Load on startup for /output clients
- Reset button: "Reset corners" sets all 4 corners back to screen edges

### T8 — Integration with Align Mode Toggle
- When align mode turns ON on /output:
  - Show corner handles
  - Apply saved CSS matrix3d() transform
  - Show Play Area boundaries (from Plan 19-1)
  - Show room overlay
- When align mode turns OFF on /output:
  - Hide corner handles
  - Keep CSS matrix3d() transform applied (projection stays calibrated)
  - Hide Play Area boundaries
  - Hide room overlay
- On non-/output clients (Dashboard/Settings): align mode toggles room overlay only (existing behavior unchanged)

## Must-haves
- 4 corner handles draggable on /output during align mode
- CSS matrix3d() transform warps the stage to match corner positions
- Arrow keys for fine-tuning (1px normal, 10px with Shift, Tab cycles corners)
- Corner positions persist to server and reload on /output startup
- Black outside the warped area
- Normal rendering when align mode is off (transform still applied from saved config)
- No impact on Dashboard/Settings view
