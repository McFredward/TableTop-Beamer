`headless: "new"` (Chromium 112+ unified-headless mode). Drop the three
Windows-specific iter15 hacks that headless-new makes irrelevant on the default
path: `--app=about:blank`, `--window-position=-32000,-32000`, `--start-fullscreen`
(already Linux-only in iter15 — confirm). Also drop `--display=` on Win32
unconditionally (it was always a no-op on Windows Chrome — there is no X server;

hatch: setting env `SSR_WIN_HEADLESS=0` reverts the headless flip to exact
iter15 headful behavior (`--app=about:blank`, `--window-position=-32000,-32000`
return; `--display=:99` remains gone — the win32 `--display=` gate is unconditional).

WIN32_ITER15_BASELINE constants. Both baseline constants (in the Wave-1 file
and in the new Wave-2 file) are updated in the same commit as the source
change. Linux baseline is UNTOUCHED.

modern Chrome's `--headless=new` supports WebRTC + getDisplayMedia + tab capture
with no behavior gap vs headful. Linux is BYTE-IDENTICAL to iter15 — guarded
by the Wave-1 snapshot test.
